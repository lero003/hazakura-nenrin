# Change: generated-index-active-links

## Changed

- Updated generated `nenrin/index.md` output to list active change ids with links to their records.
- Added a CLI test that verifies `nenrin metrics` refreshes the index with active change links.

## Reason

Habitat self-use showed that a count-only index forced an avoidable repository search before the agent could inspect active changes. Since `nenrin metrics` rewrites `index.md`, this needs to be generated by the CLI rather than maintained as a one-off ledger edit.

## Expected Behavior

- Agents can open active change records directly from `nenrin/index.md`.
- Metrics refreshes preserve active record navigation.
- Agents spend less discovery effort deciding whether an active change applies to the task.

## Review After

- 3 related task(s)
- 7 day(s)

## Success Signals

- A later task reads active change records from the index without using repository search to discover them.
- Generated index output stays short and does not duplicate full record content.

## Failure Signals

- Agents still need broad search to locate active change records after reading the index.
- The index grows into another full ledger summary instead of a navigation aid.

## Result

Unjudged.
